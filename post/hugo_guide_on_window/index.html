Windows下Hugo的安装与使用

2020年4月6日
| 阅读

需要的资源准备（前两项需自行了解）

  1. Git的下载与安装、简单git指令的熟悉与使用。

  2. Github账号及其简单的了解。

  3. Hugo安装包下载。

    地址：https://github.com/spf13/hugo/releases

安装Hugo

  1. 新建Hugo的主文件夹：D/Hugo;

  2. 新建Hugo的程序放置位置：D/Hugo/bin;

    将下载好的Hugo.zip文件中的三个文件解压至此文件夹，其中包含一个.exe的可执行文件，该文件可以在控制台中直接运行，无需安装。

  3. 将 ;D/Hugo/bin添加至系统环境变量的PATH值末尾。

    打开命令行键入hugo help,若运行成功则Hugo安装成功。（注意D/Hugo/bin下的.exe文件名若不为hugo，需改成hugo。）

新建网站

  1. 在Hugo文件夹下新建 D/Hugo/sites作为我们网站的统一放置路径。
  2. 在命令行中执行以下命令新建网站 
    C:\Users\LH> D:        //进入d盘命令
D:\> hugo              //进入hugo文件夹
D:\Hugo> cd sites      //进入sites文件夹
D:\Hugo\sites> hugo new site example.com   //新建example.com的文件夹

    若执行成功，则可以看到D:\Hugo\sites\example.com的文件夹。

  3. 为网站添加主题

    1. 将example.com文件夹初始化为git仓库

       D:\Hugo\sites> git init;       //将当前文件夹初始化 为git仓库
 Initialized empty Git repository in D:\Hugo\sites/.git/

      此处默认已安装好git环境，若未安装请自行了解。

    2. 为网站添加主题

      主题网站为：https://themes.gohugo.io/ （点击download跳转后地址栏即为主题地址）或者 https://www.gohugo.org/theme/aglaus/ 目前使用模板主题，后续主题设置更多内容请自行了解。从里面选择一个主题并找到其地址类似于 https://github.com/budparr/gohugo-theme-ananke ，其中budparr为作者名，gohugo-theme-ananke为主题名。

      git submodule add ***.git themes/ananke
//*** 为要选的主题地址，themes/annake为主题要放置的文件夹
//此处执行后从地址处下载主题，请等待下载完成。

    3. 添加内容文件

      hugo new posts/my-first-post.md

      网站内容存放在content目录下，执行成功后可以看到content/posts/my-first-post.md文件。自动帮我们生成标题。

      ---
title: "my-first-post"
date: 2020-04-06T08:58:13+08:00
draft: true
---

    4. 启动本地服务

      hugo server -t ananke --buildDrafts
//-t ananke 为指定主题为ananke。

      也可用通过修改网站路径下的config.toml文件中增加theme = “ananke"来实现指定主题。执行后可以看到如下内容。

部署到github

  1. 在github上创建仓库 创建名为name.github.io的仓库，此处为固定写法，且name为你的用户名的首字母小写。

  2. 将Hugo内容生成文件

    hugo --theme=ananke --baseUrl="地址" --buildDrafts
//同理 主题设置和url设置可以在配置文件中设置。
//此处地址为 https://name.github.io  name为你的用户名的首字母小写

    成功后会在example.com文件夹中生成public文件夹。

  3. 将public文件夹初始化为仓库并获取内容提交

    cd public      //进入public文件夹
git init       // 初始化仓库
git add .      //添加仓库中的所有内容
git commit -m "first hugo"     //-m ：为提交写备注

  4. 将public仓库更新至github仓库

    git remote add origin 地址.git    //将public仓库与远程仓库关联
//  地址为打开你刚才建的仓库时 浏览器地址栏地址（注意包括https：//）
git push -u origin master        //将public仓库更新至github仓库
//  执行成功会让你输入github用户名及密码 ，登录成功后等待上传完成即可。

  5. 访问

    在浏览器中输入 https://name.github.io 即可访问。（name为你的用户名的首字母小写）

Hugo中md文档图片的简单使用

在content文件夹下新建image文件夹，（注意不要与原public文件夹下已有的文件夹重名）将图片统一放入改文件夹,通过以下方式使用图片。

![介绍](/image/local-server.png)

//图片链接以/开头等同于baseUrl/image/local-server.png
//baseUrl为执行hugo命令生成public时指定

在生成public文件夹时会自动将content目录下image文件夹复制过来。

此方法适合在图片量较少时使用，图片太多对应的public文件夹则较大，可能造成不便。
